The "Data Type" column indicates instances where a data type flavor was used. The "Value Set" column indicates instances where a value set flavor was used. Value set information has two associated columns that are required in IGAMT: binding strength and binding location. Binding strength indicates whether the value set is required to be used (R), suggested to be used (S), or not specified (blank). Binding location indicates the location of the subcomponent where the code should be sent for complex data types. Length information is provided but is not enforceable. Also note, that a length value of "length not included" indicates a field where length was provided for a complex data type in the National IG, but length information was not included in IGAMT.
